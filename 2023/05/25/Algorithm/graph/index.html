最短路算法

常见最短路问题

源点：起点

汇点：终点

e：边数

v：顶点数

稠密图：e与v^2一个级别，适合用邻接矩阵存储

稀疏图：e与v一个级别，适合用邻接表存储

  1. 单源最短路问题

    求一个点到其他所有点的最短路问题

    单源最短路问题再分类

    1. 所有边的权重都是整数

      适用的算法：

      • 朴素版迪杰斯特拉算法：O(v^2),与边数没有关系，适用于稠密图
      • 堆优化迪杰斯特拉算法：O(elogv),适用于稀疏图，且顶点规模较大

    2. 存在负权边

      适用的算法：

      • Bellman_Ford算法：O(v*e)
      • SPFA算法：对Bellman_Ford算法进行优化，一般情况下O(e)，最坏情况下O(v*e)

  2. 多源汇最短路问题

    有多个起点，起点和终点不确定

    适用算法：

    Floyd算法：O(v^3)

朴素版Dijkstra算法

存储：邻接矩阵

初始化：起点距离dis[1]=0,其他所有点到起点的距离dis[i]=INF

集合S：表示已经确定从起点到该点的最短路径的点集，初始时为空集。

选一个还没有确定最短距离的点的最短距离，去更新其他点的距离。不断迭代，即可找到每个点的最短路。

堆优化Dijkstra算法

存储：邻接表

用堆来存储距离，维护每个点到起点的距离

Bellman-Ford算法

存储：用结构体数组存储所有的边

迭代n次循环，每次循环遍历所有的边a→b,w，更新距离dis[b] = min(dis[b],dis[a]+w)

注意：Bellman-Ford可以求有无负权回路。第n次有距离更新的话，就说明有负环。若存在负环，则不存在最短路。一般用SPFA算法判断负环。(利用鸽巢原理)

可以解决一类特殊的问题：有边数限制的最短路问题。用第一层循环次数的迭代次数来保证最多不经过多少条边。

SPFA算法

用队列存储被更新的结点，在Bellman-Ford算法的基础上进行优化，队列中存储dis[i]变小的结点。

queue←1

while(queue非空）

{

队头出队t

更新t的所有出边

如果更新成功（变小了），加入队列（还要看队列中是否已经有该节点）

}

  • spfs求最短路
  • spfa判断负环

Prim算法

  1. 朴素版prim

    ：O(n^2)，适用于稠密图

    1. 将所有点到集合的距离初始化为正无穷
    2. 用一个集合来存储生成树的结点，一开始集合为空集
    3. 迭代n次，进行如下操作
    4. 每次选取距离集合最小且不在集合的点，用这个点来更新其他点到集合的距离（该点到集合中点的距离的最小值）
    5. 将这个点加入到集合当中去
  1. 堆优化版prim：O(mlogn)，适用于稀疏图，不常用

Kruskal算法

O(mlogm)，比较常用，稀疏图一般用此算法。

执行步骤

  1. 将所有边按照权重排序（O(mlogm)）
  2. 枚举每条边，a,b,w，若a和b不连通，将这条边加入集合（并查集）。
