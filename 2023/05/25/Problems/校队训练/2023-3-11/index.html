B. Bin Packing Problem

题目描述

操作系统中内存动态分区的经典算法，first fit algorithm和best fit algorithm，给定一些内存申请序列，要求这两种算法需要的内存块数量。

题目链接

B. Bin Packing Problem

解题思路

第二个问题即first fit比较好解决，只需要用一个multiset维护当前内存块中剩余的容量，然后使用lower_bound，找到最适配的内存块，然后对其进行修改即可，若不存在，则新增一个内存块。

对于第一个问题，处理起来比较棘手，问题可以转化为，在一组无序的序列中，找到第一个大于等于给定数的位置，然后对其进行修改，可以想到支持单点修改和区间查询的线段树，线段树显然是维护区间的最大值（即最大剩余容量），定位到第一个满足要求的内存块还需要做一次二分（或者倍增），这样时间复杂度就是$O(n(\log_2(n)^2)$，对于1e6规模的数据，再加上维护线段树的常数比较大，就可能会超时。所以需要将一层$\log_2n$的操作给消掉，可以在线段树查询的过程直接查找到目标位置，查询的策略是优先在左子树节点找，这样就可以保证该位置是第一个出现的合适的位置。
