继承

Java的继承

  • 只支持单继承，每个子类只能有一个直接父类

  • 派生类继承所有组先的状态和行为

  • 派生类可以增加变量和方法

  • 派生类也可以覆盖继承的方法

基类和派生类

基类：也称为超类（superclass），是被直接或间接继承的类

派生类：也称为子类，继承其他类而得到的类。

继承的特征

  • 继承的关系具有传递性
  • 提供了软件复用功能
  • 子类不能直接访问父类的私有成员，可以通过使用共有（及保护）方法进行访问
  • 子类不能继承父类的静态属性，但是可以对父类的静态属性进行操作。

隐藏与覆盖

隐藏：子类定义同名属性

覆盖：子类重写已有的方法

属性的隐藏

  • 子类中声明了与父类中同名的属性，从父类继承的属性将会被隐藏。

  • 子类拥有两个同名的类成员变量，一个继承自父类，一个由自己申明。

  • 当子类执行继承自父类的操作时，处理的是继承自父类的变量。

  • 当子类执行它自己声明的方法时，所操作的就是它自己声明的变量。

  • 显示调用被隐藏的父类属性,super.属性名。

方法的覆盖

  • 覆盖方法的返回类型，方法名称，参数的个数及类型必须和被覆盖的方法一摸一样
  • 覆盖方法的访问权限可以比被覆盖的宽松，但是不能更为严格
  • 调用被覆盖调的父类方法，super.方法名()
  • 不能覆盖基类中声明为final的方法
  • 不能覆盖基类中声明为static的方法

继承与构造方法

  • 子类不能从父类继承构造方法
  • 可以在子类的构造函数的第一行显式地调用父类的某一个构造方法(必须在子类构造函数的第一行调用父类的构造方法)
  • 若没有在子类构造函数中显式调用父类构造方法，则会自动调用父类的无参构造方法

final修饰符

终结类

  • 被final修饰的类为终结类

  • 终结类（final类）不能被继承，即不能有派生类

终结方法

  • 被final修饰的方法为终结方法
  • 终结方法（final方法）不能被子类重写（覆盖）

abstract修饰符

抽象类

  • 被abstract所修饰的类
  • 抽象类是类层次中较高层次的概括，抽象类的作用是让其他类来继承它的抽象化的特征
  • 相当于一个标准，其他派生类必须遵循该标准
  • 不能用抽象类来创建对象

抽象方法

  • 被abstract修饰的方法

  • 只有方法头，没有方法体，具体实现在子类中重写完成

  • 抽象类可以包含抽象方法，也可以包含其他非抽象的方法，这是它与接口的不同点之一

  • 其他类若想包含抽象方法，则必须将该类声明成抽象方法

  • 抽象类中不一定包含抽象方法，但是有抽象方法的类必定是抽象类。

  • 构造方法，类方法（用 static 修饰的方法）不能声明为抽象方法。

  • 若抽象类的子类不是抽象类，则它必须重写抽象父类中的所有抽象方法

接口

接口的基本特性

  • 与抽象类一样是用来定义多个类的共同属性

  • 是纯抽象类，即只包含抽象方法（只提供形式，不提供实现）

  • 其方法默认为public abstract，允许省略修饰符

  • 其属性默认为public static final，允许省略修饰符，且必须要初始化

  • 支持多继承， 即一个接口可以有多个父接口

  • 实现接口的类必须重写所有的接口方法（包括父接口的方法）

    • 允许一个类实现多个接口

接口的扩展

  • 接口可以通过扩展的技术派生新的接口
  • 派生接口不仅保有父接口的所有成员，也可以加入新的成员
  • 实现派生接口的类也要实现其父接口
