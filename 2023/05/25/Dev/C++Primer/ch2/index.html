变量和基本类型

基本内置类型

1. 算术类型

包括整型和浮点型。

注意：在选择类型时要注意，在算术表达式中尽量不使用bool和char类型，只有在存放字符或布尔值才使用。因为char在有些机器上有符号而有些机器上是无符号的。当想要存储不大的整数时，需要明确指出使用的是signed char(-128127)还是unsigned char(0255)。

2. 空类型

void类型。

隐式类型转换

当给无符号类型赋予范围之外的值，其最终结果为取模后的值；当给有符号类型赋予范围之外的值，其结果是未定义的。这种未定义的行为依赖于运行的环境，在编写程序时应该避免这种未定义的行为。

当一个表达式中既有int又有unsigned时，int会转化为无符号值。

给unsigned赋一个负值，其最终结果为取模后的值（数学意义上的取模）。unsigned值永远都不会小于0，在使用时要注意（尤其是作为循环变量时，不过一般用int作为循环变量）。

字面值常量

浮点型字面值默认时double类型。

转义字符的泛化表示：\x跟十六进制数，或\跟八进制数。

分离式编译

c++支持分离式编译（separate compilation）机制，该机制允许将程序分为若干个文件，每个文件可以被独立编译。为了支持这种机制，c++必须将声明和定义区分开来。声明是的名字为程序所知；而定义负责创建与名字关联的实体。任何包含显示初始化的声明即成为定义。

如果想声明一个变量，就在前面加上关键字extern，并且不要有显示的初始化，否则编译器会报错。

注：一个变量只能被定义一次，但是可以被多次声明。

标识符

由字母、下划线、数字构成，且只能以字母或者下划线开头。

名字的作用域

名字的有效区域始于名字的声明语句，以声明语句所在的作用域末端作为结束。

允许在内层域中定义外层域中已有的名字。

复合类型

  1. 引用：相当于给对象起的别名，实际上操作的还是原对象。即定义了一个引用之后，对其进行的所有操作都是在与之绑定的对象上进行的。

    注意：

    1. 引用必须被初始化，否则编译器会报错。

    2. 引用定义时，程序会把引用和它的初始值绑定在一起。因为引用无法重新绑定到另外一个对象，因此引用必须初始化。

    3. 因为引用本身并不是一个对象，因此不存在引用的引用。

  2. 指针：是指向另外一种类型的复合类型。与引用的类似之处在于，指针也实现了对其他对象的间接访问。指针与引用的不同之处如下：

    1. 指针本身就是一个对象，允许对指针进行赋值和拷贝

    2. 在指针的生命周期内，允许指向多个对象。

    3. 指针无需在定义时初始化。和其他内置类型一样，在块作用域内定义的指针若没有被初始化将拥有一个不确定的值。

    注意：

    1. int *p = &i 为指针变量常见的定义方式，其中&为取地址符。通过*号来访问指针所指的对象，被称为解引用符。指针实际存储的是对象的地址。

    2. 除了一些特例外，指针所指向对象的类型，必须与定义指针时的类型一致。

    3. 给解引用的指针赋值，实际上就是对指针所指向的对象赋值。

    const 限定符

    1. const 用于定义常量，常量一旦被定义，就不能再修改，所以常量必须被初始化。

    2. 默认情况下，const 对象仅在文件内有效，若想在多个文件之间共享const 对象，则必须在变量的定义之前加extern关键字。

    3. const的引用，将绑定到const 对象的引用称为对常量的引用。

    4. 常量引用：注意，允许将常量引用绑定一个非常量的对象、字面值。以下定义都是合法的。即初始化常量引用时，允许用任意表达式作为初始值。若常量引用绑定了一个非常量的对象，就不允许通过改引用去修改对象的值。

    5. 指向常量的指针：不能用该指针去改变所指对象的值。若想存档常量对象的地址，只能使用指向常量的指针。但允许指向常量的指针指向一个非常量。这点和常量引用一样，即指向常量的指针也没有规定其所指的对象必须是一个常量。而所谓指向常量的指针，仅要求不能通过该指针改变所指对象的值。

    1. const指针：指针是对象，而引用不是，因此允许把指针本身定义为常量，常量指针必须初始化，一旦确定了所指的对象，就不能再修改。

    2. 顶层const：顶层const（top-level const）表示指针本身是一个常量，而用底层const（low-level const）表示指针所指的对象是一个常量。指针可以同时为顶层const和底层const，这一点和其他类型有明显区别。执行对象的拷贝操作时，顶层const不受什么影响。底层const的限制不能忽略，当执行对象的拷贝操作时，拷入和拷出的对象必须具有相同的底层const资格，或者两个对象的数据类型可以相互转换，一般来说非const可以转换成const，反之则不行。不能用指向const对象的给非底层const的指针赋值。

    3. 常量表达式和constexpr：常量表达式是值不会改变，并且在编译过程中就能得到计算结果的表达式，一个对象是不是常量表达式由它的数据类型和初始值共同决定。c++11允许使用constexpr类型，让编译器来验证变量的值是否是一个常量表达式。声明为constexpr的变量一定是一个常量，而且必须使用常量表达式初始化。

    4. constexpr与指针：constexpr类型的指针一般用来指向存放在固定地址中的对象（一般是定义在所有函数体之外的对象的地址），constexpr仅对指针有效，即consexpr int* p = nullptr，定义了一个常量指针，而非指向常量对象的指针。

    处理类型

    类型别名：使用typedef 给类型区别名，方便记忆和编写。

    还可以使用using 来给类型取别名。

    auto类型说明符：使用auto 类型说明符能够让编译器替我们去分析表达式所属的类型。使用auto在一条语句中申明多个变量时，要求所有变量的初始数据类型一样。

    注意：

    1. 初始化auto类型的对象时使用引用类型的对象，则实际上使用的引用所绑定对象的值。

    2. auto一般会忽略顶层const，但会保留底层const。若想推出的类型是一个顶层const，需明确指出const auto。

    decltype类型指示符：作用是选择并返回操作数的数据类型。decltype((variable))的结果永远都是引用（双层括号），decltype(variable)只有当variable本身是引用时其结果才是引用。
